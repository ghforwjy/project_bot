# 前端开发

## 1. 技术栈与架构

### 1.1 技术栈

| 技术 | 版本 | 用途 | 选型理由 |
|------|------|------|----------|
| **React** | 18.x | UI框架 | 生态成熟，组件化开发，社区活跃 |
| **TypeScript** | 5.x | 类型系统 | 强类型保证，提升代码质量和可维护性 |
| **Vite** | 5.x | 构建工具 | 快速冷启动，HMR支持，配置简单 |
| **Ant Design** | 5.x | UI组件库 | 企业级组件，表格/表单功能强大 |
| **Zustand** | 4.x | 状态管理 | 轻量级，TypeScript友好，学习成本低 |
| **React Query** | 5.x | 服务端状态 | 数据获取、缓存、同步管理 |
| **ECharts** | 5.x | 甘特图绘制 | 功能强大，支持自定义，文档完善 |
| **Axios** | 1.x | HTTP客户端 | 拦截器支持，请求取消，TypeScript支持 |

### 1.2 前端架构

前端采用 **React + TypeScript** 技术栈，使用 **Zustand** 进行状态管理，**Ant Design** 作为UI组件库，**ECharts** 用于甘特图绘制。

## 2. 组件设计

### 2.1 布局组件
- **MainLayout**: 整体页面布局，包含顶部导航、左侧聊天区域、中央项目区域、右侧详情侧边栏
- **ChatPanel**: 聊天区域组件，包含消息列表和输入框
- **ProjectPanel**: 项目信息区域，包含项目列表和甘特图

### 2.2 聊天组件
- **ChatWindow**: 聊天窗口，管理消息展示和滚动
- **MessageList**: 消息列表，展示用户和AI的对话
- **MessageItem**: 单个消息项，支持不同角色的消息样式
- **InputBox**: 输入框，支持文本输入和发送

### 2.3 项目组件
- **ProjectTable**: 项目表格，展示项目列表和基本信息
- **TaskDetail**: 任务详情，展示任务的详细信息
- **GanttChart**: 甘特图组件，可视化展示项目进度

### 2.4 配置组件
- **ConfigModal**: 配置模态框，用于配置LLM提供商和API Key
- **ModelSelector**: 模型选择器，选择不同的LLM模型

## 3. 状态管理

### 3.1 状态管理方案
使用 **Zustand** 进行状态管理，按功能模块划分不同的store：

- **chatStore**: 管理聊天相关状态，包括消息列表、会话信息等
- **projectStore**: 管理项目相关状态，包括项目列表、任务信息等
- **configStore**: 管理配置相关状态，包括LLM配置、API Key等

### 3.2 状态持久化
- 使用 localStorage 存储配置信息
- 聊天历史和项目数据通过API与后端同步

## 4. API服务

### 4.1 API封装
使用 **Axios** 封装API调用，提供统一的错误处理和响应格式：

- **chatApi**: 聊天相关API，包括发送消息、获取历史记录等
- **projectApi**: 项目相关API，包括项目CRUD、任务管理等
- **configApi**: 配置相关API，包括获取和更新配置等

### 4.2 实时通信
使用 **WebSocket** 实现实时通信，支持：
- 流式对话响应
- 项目信息实时更新
- 任务状态变更通知

## 5. UI/UX设计

### 5.1 设计原则
- **简洁明了**: 界面简洁，操作直观
- **响应式设计**: 适配不同屏幕尺寸
- **一致性**: 保持设计风格一致
- **可访问性**: 符合WCAG标准

## 6. 开发指南

### 6.1 开发环境搭建

#### 步骤1: 安装依赖
```bash
cd frontend
npm install
```

#### 步骤2: 启动开发服务器
```bash
npm run dev
```

#### 步骤3: 构建生产版本
```bash
npm run build
```

### 6.2 代码规范
- 使用 TypeScript 类型定义
- 遵循 ESLint 规范
- 使用 Prettier 格式化代码
- 组件命名使用 PascalCase
- 文件命名使用 kebab-case

### 6.3 测试策略
- 使用 Jest 进行单元测试
- 使用 React Testing Library 进行组件测试
- 集成测试覆盖主要业务流程

## 7. 性能优化

### 7.1 优化策略
- **代码分割**: 使用动态导入减少初始加载时间
- **缓存策略**: 合理使用 React Query 缓存
- **渲染优化**: 使用 React.memo、useCallback、useMemo
- **网络优化**: 合理使用 HTTP 缓存和压缩

### 7.2 监控指标
- **首屏加载时间**
- **交互响应时间**
- **内存使用情况**
- **API 请求性能**
