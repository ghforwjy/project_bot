# LangChain意图识别与多轮对话系统需求文档

## 1. 项目概述

本项目旨在构建一个基于LangChain和LangGraph的智能项目管理助手，通过自然语言交互帮助用户管理项目、任务和项目大类。系统应具备准确的意图识别、智能的多轮对话能力、以及与后端业务逻辑的无缝集成。

## 2. 核心功能需求

### 2.1 意图识别与分类
- **功能**：能够准确识别用户输入的意图，包括创建、更新、删除、查询项目/任务/项目大类等操作
- **技术**：使用LangChain框架构建意图分类器，结合对话历史提高识别准确性
- **架构**：采用LangGraph构建有状态的对话系统，管理对话历史和状态
- **组合任务**：能够识别包含多个操作的复杂请求，将其拆分为多个子任务

### 2.2 多轮对话支持
- **功能**：能够理解上下文信息，处理用户的指代关系（如"他"、"这个"等）
- **技术**：使用LangGraph的状态管理能力，维护对话历史和实体指代关系
- **流程**：通过LangGraph的节点和边定义对话流程，处理用户的后续问题和确认

### 2.3 业务逻辑执行
- **功能**：根据识别的意图执行相应的业务逻辑，如创建项目、更新任务状态等
- **技术**：通过LangChain的ToolCalling能力，将意图与后端业务逻辑连接
- **集成**：与现有的项目管理系统API集成，执行实际的业务操作
- **组合任务执行**：能够按顺序执行多个子任务，收集执行结果并生成综合响应

### 2.4 自然语言响应
- **功能**：以自然、友好的语言向用户反馈操作结果
- **技术**：使用LLM生成自然语言回复，确保回复符合上下文语境
- **格式**：回复应简洁明了，包含必要的信息，避免技术术语

### 2.5 项目不存在处理
- **功能**：当用户提及的项目不存在时，应查找相似项目并提供建议
- **技术**：实现简单的字符串相似度匹配算法，查找可能的相似项目
- **流程**：如果找到相似项目，询问用户是否指的是该项目；如果没有找到，显示系统中所有可用项目

## 3. 技术架构

### 3.1 核心框架
- **LangChain Core**：作为基础框架，提供与LLM的交互能力
- **LangGraph**：用于构建有状态的对话流程，管理对话历史和状态
- **LCEL (LangChain Expression Language)**：用于组合不同组件，构建处理管道
- **LangChain Agents**：用于处理复杂的决策逻辑，如需要查询数据库或执行操作时

### 3.2 组件设计
- **意图分类器**：使用LangChain的PromptTemplate和LLMChain构建，结合对话历史提高识别准确性
- **对话状态管理器**：使用LangGraph的状态管理能力，维护对话历史和实体指代关系
- **数据提取器**：使用LangChain的OutputParser从LLM响应中提取结构化数据
- **业务逻辑执行器**：通过LangChain的ToolCalling能力，将意图与后端业务逻辑连接

### 3.3 数据流
1. 用户输入 → 意图分类器 → 识别意图和提取数据
2. 意图和数据 → 对话状态管理器 → 处理上下文和指代关系
3. 处理后的意图和数据 → 业务逻辑执行器 → 执行相应的业务操作
4. 操作结果 → LLM → 生成自然语言回复
5. 回复 → 用户

## 4. 技术实现要点

### 4.1 意图分类
- 使用PromptTemplate定义意图分类的提示，包含所有可能的意图类型
- 结合对话历史，提高意图识别的准确性
- 使用OutputParser将LLM的响应解析为结构化的意图数据

### 4.2 多轮对话
- 使用LangGraph构建对话状态机，包含意图识别、实体提取、操作执行等节点
- 定义状态更新逻辑，维护对话历史和实体信息
- 处理用户的指代关系，如"他"、"这个"等

### 4.3 业务逻辑集成
- 将后端业务逻辑封装为工具，通过Agent执行
- 处理工具执行的结果，生成自然语言回复
- 处理异常情况，提供友好的错误提示

### 4.4 项目不存在处理
- 实现字符串相似度匹配算法，查找可能的相似项目
- 生成友好的提示，询问用户是否指的是相似项目
- 如果没有相似项目，显示系统中所有可用项目
