# 问题分析

通过分析代码，我发现了任务编辑弹框显示在项目详情弹框后面的原因：

1. **层级设置相同**：
   - `ProjectDetailModal` 和 `TaskModal` 都使用了相同的 z-index 值
   - 背景遮罩：两者都是 `zIndex: 1000`
   - 内容容器：两者都是 `zIndex: 1001`

2. **渲染顺序问题**：
   - `TaskModal` 是在 `ProjectDetailModal` 内部渲染的
   - 根据 CSS 层叠上下文规则，即使设置了相同的 z-index，后渲染的元素也会被先渲染的元素遮挡

# 解决方案

为了确保任务编辑弹框显示在项目详情弹框的上方，需要调整 `TaskModal` 的层级值：

1. **修改 TaskModal.tsx**：
   - 将背景遮罩的 z-index 从 1000 改为 1002
   - 将内容容器的 z-index 从 1001 改为 1003

2. **保持 ProjectDetailModal.tsx 不变**：
   - 维持原有的层级设置，确保不影响其他功能

# 预期效果

修改后，当在项目详情页面中点击任务的编辑按钮时，任务编辑弹框将会正确显示在项目详情弹框的上方，用户可以正常编辑任务而不会被遮挡。
