# 修复任务列表编辑功能的错误

## 问题分析

根据错误信息和代码分析，发现以下问题：

1. **组件类型无效错误**：错误指向ProjectDetailModal.tsx的第279行，显示"Element type is invalid: expected a string but got: undefined"

2. **Ant Design组件使用错误**：代码中使用了`Input.Number`组件，但在Ant Design中正确的组件名称应该是`InputNumber`（独立组件，没有点号）

3. **废弃属性警告**：Dropdown组件使用了已废弃的`overlay`属性，应该使用`menu`属性

## 修复计划

### 1. 修复ProjectDetailModal.tsx中的组件导入和使用

- 在导入语句中添加`InputNumber`组件
- 将所有`Input.Number`替换为`InputNumber`

### 2. 检查并修复ProjectPanel.tsx中的类似问题

- 检查是否也使用了`Input.Number`组件
- 如有，同样替换为`InputNumber`并添加相应导入

### 3. 修复Dropdown组件的废弃属性

- 将Dropdown组件的`overlay`属性替换为`menu`属性

### 4. 验证修复结果

- 重新启动前端开发服务器
- 测试任务列表的编辑功能是否正常工作
- 确保所有错误和警告都已消除

## 技术细节

- Ant Design版本中，`InputNumber`是独立导出的组件，不是`Input`的子组件
- Dropdown组件的`overlay`属性已在新版本中废弃，推荐使用`menu`属性
- 修复后需要确保所有组件都能正确渲染，特别是编辑状态下的表单控件

## 预期结果

- 任务列表的编辑功能正常工作
- 所有React组件类型错误消除
- 所有Ant Design废弃属性警告消除
- 前端应用能够正常启动和运行
