## 问题分析

在项目详情页面中，任务的时间控件清空后一保存就还原的问题，经过分析发现：

1. **前端逻辑**：当用户清空日期选择器时，前端会将日期值设置为 `undefined` 并发送到后端
2. **后端逻辑**：后端在处理日期字段时，只处理有值的情况，当日期字段为 `None` 时不会更新该字段
3. **根本原因**：后端的 `update_task` 函数中，日期处理逻辑使用了 `if field in update_data and update_data[field]:` 条件，当 `update_data[field]` 为 `None` 时条件不成立，不会更新该字段

## 解决方案

修改后端的任务更新逻辑，使其能够处理日期字段为 `None` 的情况，即将字段设置为 `None`。

## 修改步骤

1. **修改后端任务更新逻辑**：在 `backend/api/task.py` 文件中，修改 `update_task` 函数的日期处理逻辑
2. **更新日期字段处理**：将日期字段的处理逻辑改为，无论是否有值，只要在 `update_data` 中存在就更新
3. **保持其他逻辑不变**：确保其他字段的处理逻辑和项目概要更新逻辑不受影响

## 技术实现

将后端代码中的日期处理逻辑从：
```python
for field in date_fields:
    if field in update_data and update_data[field]:
        update_data[field] = datetime.fromisoformat(update_data[field])
```

修改为：
```python
for field in date_fields:
    if field in update_data:
        if update_data[field]:
            update_data[field] = datetime.fromisoformat(update_data[field])
        else:
            update_data[field] = None
```

这样，当前端发送 `None` 值时，后端会将对应字段设置为 `None`，实现清空日期的功能。
