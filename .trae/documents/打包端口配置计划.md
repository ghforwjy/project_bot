# 打包端口配置计划

## 目标
打包后的程序使用可配置端口，默认 8118（后端），5118（前端），不占用 8000 和 5173 端口。

## 需要修改的内容

### 1. 修改端口配置逻辑
- 后端端口：从 .env 读取 `PORT`（默认 8118）
- 前端端口：从 .env 读取 `FRONTEND_PORT`（默认 5118）
- 本地开发环境（未打包）保持原有端口不变

### 2. 修改 .env 文件
添加端口配置：
```
PORT=8118
FRONTEND_PORT=5118
```

### 3. 修改前端配置
- 前端 vite 配置读取环境变量中的端口

### 4. 更新打包脚本
- 打包时包含 data 目录和 .env 文件
- .env 中设置默认端口为 8118/5118

### 5. 验证打包结果
- 确认 release_standalone 包含 data 目录和 .env
- 测试运行是否使用 8118 端口

## 执行顺序
1. 修改 backend/main.py - 添加端口读取逻辑
2. 修改 frontend/vite.config.ts - 添加端口配置
3. 修改 .env - 添加端口变量
4. 更新打包脚本 build_nuitka.py
5. 重新打包 standalone 版本
6. 验证端口配置
